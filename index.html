About this prototype. The gauge uses a physics-informed approximation (irradiance × capacity, derated ~0.4%/°C above 25°C module temperature, per standard PV temperature-coefficient behavior) rather than the notebook's trained RandomForestRegressor — that model runs in Python and would need to be served from a small backend (e.g. Flask/FastAPI, or wrapped in Streamlit as sketched at the end of the notebook) for a browser UI to call it directly. Live readings use the same Open-Meteo geocoding + forecast endpoints as the notebook's fetch_city_live_power cell.
